# Demo Makefile Target and Transcript Formatter

**Task ID:** sprint-10/spec-03-demo-make-target-and-transcript
**Date:** 2026-05-11

## Overview

Replaces the stub `demo` target in the `Makefile` with a real target that cleans stale artifacts, runs the demo driver (`tools/demo.mjs`), and writes `demo/transcript.md` — a committed Markdown artifact capturing each demo step's input payload and hook output. Extends `tools/demo.mjs` with a `writeTranscript({path, steps})` function that formats the four demo steps into `## Step N` sections with fenced code blocks and appends a verbatim caveat block about the next-turn scrubber limitation.

## What Was Built

- `demo` Makefile target: creates `demo/`, removes stale artifacts, runs the driver with `CLAUDE_PLUGIN_DATA=$(CURDIR)/demo/`, writes the transcript, prints a success summary
- `clean-demo` Makefile target: removes `demo/audit.jsonl` and `demo/transcript.md`
- `writeTranscript({path, steps})` function in `tools/demo.mjs`: serialises step results to Markdown (four `## Step N` sections + `## Caveat` blockquote)
- `.gitignore` updated: `demo/audit.jsonl` ignored; `demo/transcript.md` remains tracked
- `demo/transcript.md` committed as a static launch artifact

## Technical Implementation

### Files Modified

- `Makefile`: Added `clean-demo` to `.PHONY`, replaced stub `demo` target with real implementation, added `clean-demo` target; also extended `validate` to lint `marketplace.json`
- `tools/demo.mjs`: Extended with `writeTranscript` function and `--write-transcript=<path>` flag wiring
- `.gitignore`: Replaced `tools/` ignore entry with `demo/audit.jsonl`

### New Files

- `demo/transcript.md`: Generated by `make demo`; committed to the repository as a launch artifact

### Key Changes

- The `demo` target sets `CLAUDE_PLUGIN_DATA=$(CURDIR)/demo/` as an inline environment variable prefix so the audit log lands in `demo/audit.jsonl` rather than `~/.claude/sentinel/audit.jsonl`, keeping the demo hermetic.
- `writeTranscript` uses only `node:fs` (`writeFileSync`) and `node:path` (`resolve`) — no new npm dependencies. It is wrapped in `try { … } catch (err) { process.stderr.write(...) }` so a write failure emits a warning to stderr but does not affect the driver's exit code (fail-open).
- Each step element has the shape `{ label, input, output }`. Step 4 (review-cli) populates `input` with the command string rather than a JSON payload.
- The `## Caveat — next-turn scrubber` section reproduces verbatim the blockquote prose from `README.md:22`, satisfying the honest-limitation requirement from the research doc.
- `demo/audit.jsonl` is gitignored; `demo/transcript.md` is not. This ensures the transcript can be embedded by the README without requiring a live `make demo` run, while keeping the ephemeral JSONL local to each developer's machine.

## How to Use

1. Run the demo end-to-end:
   ```sh
   make demo
   ```
   Produces `demo/transcript.md` and prints `demo: ok — transcript written to demo/transcript.md`.

2. Clean demo artifacts before a fresh run:
   ```sh
   make clean-demo && make demo
   ```
   The second run produces a structurally identical `demo/transcript.md` (ULID/timestamp values in `audit.jsonl` differ; transcript headings and code block shapes are deterministic).

3. Inspect the transcript:
   ```sh
   cat demo/transcript.md
   ```
   Contains four `## Step N — <description>` sections and a `## Caveat — next-turn scrubber` section.

4. Run without writing a transcript (driver-only mode):
   ```sh
   node tools/demo.mjs
   ```

## Configuration

- `CLAUDE_PLUGIN_DATA` — set by the `demo` Makefile target to `$(CURDIR)/demo/` to redirect the audit log. Do not export this variable globally; the Makefile sets it inline per invocation.
- `--write-transcript=<path>` — CLI flag consumed by `tools/demo.mjs`. When absent, the driver runs and reports step results but writes no file.

`tests/demo.test.mjs` covers subprocess integration: asserts exit 0, correct step output patterns, and that `demo/transcript.md` contains all five expected headings after a `--write-transcript` run.

## Notes

- `writeTranscript` lives in `tools/demo.mjs` (not a separate file) because the formatter is tightly coupled to the step result shape defined in the driver and is ~50 lines. Splitting it would add a cross-file import with no encapsulation benefit.
- The `demo` and `clean-demo` targets follow the `validate` target style (`Makefile:3-9`): tab indentation, `@`-silenced commands, a final `@echo` success line.
- The caveat wording must come verbatim from `README.md:22` — not paraphrased — because the research doc requires the scrubber limitation to be "surfaced honestly in the demo output."
